# 推广中心

推广数据查询、排行榜、渠道统计相关接口。通过 `openApi.*` 命名空间访问全站推广概览数据，通过 `referral.*` 命名空间管理自己的推广链接、查看个人推广统计和签到领奖。

---

## 概览数据（openApi）

以下端点通过 `openApi` 路由提供，需要 `referral:read` scope。

### 推广数据总览

<Endpoint method="POST" path="openApi.referralOverview" auth="referral:read" />

获取全站推广活动的汇总数据，包含总链接数、总邀请人数、点击量、付款统计等。支持按日期范围和渠道筛选，不传参数则返回全量汇总。

<ParamTable>
  <P name="from" type="string">开始日期（ISO 8601 格式），与 `to` 一起使用，启用日期范围筛选</P>
  <P name="to" type="string">结束日期（ISO 8601 格式，范围不超过 90 天）</P>
  <P name="channel" type="string">渠道标识，仅统计指定渠道的链接数据</P>
</ParamTable>

**示例 1 — 全量汇总：**

<CopyBlock>{`curl -X POST 'https://your-domain.com/api/trpc/openApi.referralOverview' \\
  -H 'Content-Type: application/json' \\
  -H 'Authorization: Bearer sk-your-api-key' \\
  -d '{}'`}</CopyBlock>

**示例 2 — 指定日期范围 + 渠道：**

<CopyBlock>{`curl -X POST 'https://your-domain.com/api/trpc/openApi.referralOverview' \\
  -H 'Content-Type: application/json' \\
  -H 'Authorization: Bearer sk-your-api-key' \\
  -d '{"from":"2026-03-01T00:00:00.000Z","to":"2026-03-31T23:59:59.999Z","channel":"twitter"}'`}</CopyBlock>

**返回字段说明：**
- `period` — 筛选的日期范围，未指定时为 `null`
- `channel` — 筛选的渠道，未指定时为 `null`
- `links` — 总链接数
- `referrals` — 总邀请记录数
- `clicks` / `uniqueClicks` — 总点击 / 去重点击
- `payments.count` / `payments.amount` — 付款笔数 / 金额
- `pointsAwarded` — 已发放推广积分
- `activeReferrers` — 活跃推广者数

### 推广趋势统计

<Endpoint method="POST" path="openApi.referralTrendStats" auth="referral:read" />

按日聚合的全站推广数据时间序列，适用于趋势图表可视化。支持渠道筛选。

<ParamTable>
  <P name="from" type="string" required>开始日期（ISO 8601 格式）</P>
  <P name="to" type="string" required>结束日期（ISO 8601 格式，范围不超过 90 天）</P>
  <P name="channel" type="string">渠道标识，仅统计指定渠道的数据</P>
</ParamTable>

<CopyBlock>{`curl -X POST 'https://your-domain.com/api/trpc/openApi.referralTrendStats' \\
  -H 'Content-Type: application/json' \\
  -H 'Authorization: Bearer sk-your-api-key' \\
  -d '{"from":"2026-03-01T00:00:00.000Z","to":"2026-03-15T23:59:59.999Z"}'`}</CopyBlock>

**返回字段说明：**
- `period.from` / `period.to` — 查询的日期范围
- `channel` — 筛选的渠道，未指定时为 `null`
- `trend[]` — 每日数据数组，每项包含 `date`、`clicks`、`uniqueClicks`、`registers`、`paymentCount`、`paymentAmount`

### 推广排行榜

<Endpoint method="POST" path="openApi.referralLeaderboard" auth="referral:read" />

按不同指标查看推广用户排行榜。支持渠道筛选（`points` 指标除外）。

<ParamTable>
  <P name="metric" type="string">排行指标：`referrals`（邀请人数，默认）/ `clicks`（点击量）/ `points`（积分）</P>
  <P name="limit" type="number">数量（1-50，默认 20）</P>
  <P name="channel" type="string">渠道标识，仅统计指定渠道的链接数据（`points` 指标时无效）</P>
</ParamTable>

<CopyBlock>{`curl -X POST 'https://your-domain.com/api/trpc/openApi.referralLeaderboard' \\
  -H 'Content-Type: application/json' \\
  -H 'Authorization: Bearer sk-your-api-key' \\
  -d '{"metric":"referrals","limit":10,"channel":"bilibili"}'`}</CopyBlock>

**返回字段说明：**
- `items[].userId` — 用户 ID
- `items[].name` — 用户昵称
- `items[].avatar` — 头像 URL
- `items[].value` — 指标数值

### 推广渠道统计

<Endpoint method="POST" path="openApi.referralChannelStats" auth="referral:read" />

按推广渠道（channel）聚合统计点击、注册、付款数据，包含转化率计算。支持日期范围筛选和自定义排序。

<ParamTable>
  <P name="limit" type="number">数量（1-50，默认 20）</P>
  <P name="from" type="string">开始日期（ISO 8601 格式），与 `to` 一起使用</P>
  <P name="to" type="string">结束日期（ISO 8601 格式，范围不超过 90 天）</P>
  <P name="sortBy" type="string">排序字段：`registers`（默认）/ `clicks` / `uniqueClicks` / `payments`</P>
</ParamTable>

**示例 — 查询本月各渠道表现：**

<CopyBlock>{`curl -X POST 'https://your-domain.com/api/trpc/openApi.referralChannelStats' \\
  -H 'Content-Type: application/json' \\
  -H 'Authorization: Bearer sk-your-api-key' \\
  -d '{"from":"2026-03-01T00:00:00.000Z","to":"2026-03-31T23:59:59.999Z","sortBy":"registers","limit":10}'`}</CopyBlock>

**返回字段说明：**
- `period` — 筛选的日期范围，未指定时为 `null`
- `items[].channel` — 渠道名称
- `items[].clicks` / `uniqueClicks` — 点击 / 去重点击
- `items[].registers` — 注册数
- `items[].payments` / `paymentAmount` — 付款笔数 / 金额
- `items[].conversionRate` — 注册转化率（百分比）

### 推广每日报表

<Endpoint method="POST" path="openApi.referralDailyReport" auth="referral:read" />

按日期×渠道聚合的推广数据报表，每行代表一个日期和渠道的组合，包含当日数据、累计值和积分。适合导出到表格或构建数据看板。支持按链接 ID 和渠道筛选。

<ParamTable>
  <P name="from" type="string" required>开始日期（ISO 8601 格式）</P>
  <P name="to" type="string" required>结束日期（ISO 8601 格式，范围不超过 90 天）</P>
  <P name="linkIds" type="string[]">推广链接 ID 数组，仅统计指定链接的数据</P>
  <P name="channel" type="string">渠道标识，仅统计指定渠道的数据</P>
</ParamTable>

**示例 — 查询 3 月全量报表：**

<CopyBlock>{`curl -X POST 'https://your-domain.com/api/trpc/openApi.referralDailyReport' \\
  -H 'Content-Type: application/json' \\
  -H 'Authorization: Bearer sk-your-api-key' \\
  -d '{"from":"2026-03-01T00:00:00.000Z","to":"2026-03-31T23:59:59.999Z"}'`}</CopyBlock>

**示例 — 按链接筛选：**

<CopyBlock>{`curl -X POST 'https://your-domain.com/api/trpc/openApi.referralDailyReport' \\
  -H 'Content-Type: application/json' \\
  -H 'Authorization: Bearer sk-your-api-key' \\
  -d '{"from":"2026-03-01T00:00:00.000Z","to":"2026-03-31T23:59:59.999Z","linkIds":["link_id_1","link_id_2"]}'`}</CopyBlock>

**返回字段说明：**
- `period.from` / `period.to` — 查询的日期范围
- `rows[]` — 每日×渠道数据行，每项包含：
  - `date` — 日期（`YYYY-MM-DD`）
  - `channel` — 渠道标识，无渠道时为 `null`
  - `clicks` — 当日点击数
  - `registers` — 当日注册数
  - `cumulativeClicks` — 截至该日的累计总点击
  - `cumulativeRegisters` — 截至该日的累计总注册
  - `points` — 当日推广积分奖励

  -H 'Authorization: Bearer sk-your-api-key' \\

## 用户推广管理（referral）

以下端点通过 `referral` 路由提供，操作当前登录用户自己的推广数据。查询类需要 `referral:read` scope，写入类需要 `referral:write` scope。

### 获取我的推广统计

<Endpoint method="POST" path="referral.getMyStats" auth="referral:read" />

查询当前用户的推广统计信息，包含指定日期、前一日、全量汇总和转化率。可选指定日期和链接 ID 筛选。

<ParamTable>
  <P name="date" type="string">查询日期（`YYYY-MM-DD` 格式，如 `2026-03-15`），不传则为今日</P>
  <P name="linkIds" type="string[]">推广链接 ID 数组，仅统计指定链接的数据</P>
</ParamTable>

**示例 — 查询指定日期的统计：**

<CopyBlock>{`curl -X POST 'https://your-domain.com/api/trpc/referral.getMyStats' \\
  -H 'Content-Type: application/json' \\
  -H 'Authorization: Bearer sk-your-api-key' \\
  -d '{"date":"2026-03-15"}'`}</CopyBlock>

**返回字段说明：**
- `date` — 查询的日期（`YYYY-MM-DD`）
- `points` — 当前积分余额
- `referralCode` — 个人推广码
- `totalReferrals` / `totalLinks` — 总邀请人数 / 总链接数
- `todayClicks` / `todayUniqueClicks` / `todayRegisters` / `todayPaymentCount` / `todayPaymentAmount` — 指定日期的数据
- `yesterdayUniqueClicks` / `yesterdayRegisters` / `yesterdayPaymentCount` — 指定日期前一天的数据
- `totalClicks` / `totalUniqueClicks` / `totalRegisters` / `totalPaymentCount` / `totalPaymentAmount` — 全量数据
- `conversionRate` — 注册转化率（百分比）
- `paymentConversionRate` — 付款转化率（百分比）
- `earnedPoints` — 推广获得的总积分
